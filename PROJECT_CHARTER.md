## 1. 왜 만드는가

학교 현장은 무분별한 아동학대 고소·고발에 노출되어 있고, 교사에게는 *방패*가 없다. 변호사를 학교에 두는 것이 이상적이지만 현실적으로 어려우므로, 변호사의 핵심 역할 중 하나인 *"이 사안은 법적으로 어떤 의미인지 알려주는 것"* 을 AI 기반 도구로 대체한다.

핵심 사용자는 *아동학대 신고를 받았거나 받을 가능성에 직면한 교사*. 위기 상황에서 *3분 안에 자기 사안과 유사한 판례를 찾고, 법원의 판단 경향을 이해하고, 다음 행동을 결정할 수 있도록* 돕는다.

**개발 목적은 수익화가 아니라 교사들에게 실질적으로 도움이 되는 것이다.** 이 원칙이 모든 결정의 출발점이다.

---

## 2. 서비스 정체성

**우리는 법률 자문이 아니라 판례 탐색·정보 제공 서비스다.** 변호사법 109조 저촉 위험 회피를 위해 이 포지셔닝을 모든 화면·문구·면책 고지에 일관되게 표현한다.

핵심 가치는 *"교사가 직접 찾기 어려운 판례를 쉽게 찾고 이해하도록 돕는 도구"*. 단정적 답을 주는 게 아니라 *법원이 어떻게 판단해왔는지의 경향을 보여주는 자료실*이다.

**무료라고 책임이 줄어들지 않는다.** 환각 방어와 면책 고지는 유료 서비스보다 더 철저하게 적용된다.

## 3. 사용자 흐름 (Phase 1 구현 완료)

5+2 화면 중 **핵심 3 + 부속 2 구현 완료**.

1. **랜딩** (`/`) — 신뢰 카드 3개(무료·익명·실시간 검증) + AI 환각 차별점 강조 박스 + CTA
2. **사안 입력** (`/analyze`) — 체크리스트(행위·시점·학년) + **위험 신호 체크박스**(반복성·신체 피해·관련 학생 수) + 자유 서술
3. **로딩** (`/report/[id]` 진입 시) — 4단계 진행 표시 + 검증된 출처 카드 + 법률 상식
4. **분석 리포트** (`/report/[id]`) — 헤드라인·통계·요인·판례 카드 N건·대응 체크리스트 + PDF 출력
5. **판례 상세** — Phase 1 미구현. Phase 2에서 판례 카드 → 상세 페이지 전환 예정

**분기 화면 2개**
- **위험 신호 화면** — 리포트 페이지 내 *노란 배너*로 통합 (헌장 v3는 별도 화면, Phase 1은 인라인). 체크박스 기반 결정론적 판정
- **법률지원 채널** (`/support`) — 전국 5개 채널 + tel: 링크. 위험 배너에서 직접 연결

- **세션 데이터** — 사용자 입력은 `sessionStorage`로 전달, 분석 직후 폐기. 결과는 같은 sessionId 캐시
- **위험 신호** — **사용자 체크박스로 결정론적 판정** (v3는 LLM 추정이었으나 환각 모순 학습 후 폐기)

**판례는 저장하지 않음** — 법제처 OPEN API에서 매번 실시간 호출. 분석은 LLM이 그 자리에서 수행.

## 5. 핵심 기술 흐름 (실증 검증·구현 완료)

분석 1회는 다음 3단계 + 법제처 API + **사건명 prefilter** + **유사도 임계치**.

### 프롬프트 A — 키워드 변환
- `extractKeywords()` (`lib/extractKeywords.ts`), `responseSchema` JSON 강제
- 일상 용어 → 법률 키워드 매핑 표는 v3와 동일
- **v3 변경**: riskFlag·riskReason 제거 (위험 신호는 사용자 체크박스 기반으로 이동)

### 법제처 API 호출 — 4단계 확장 + 사건명 prefilter

기본 호출 형식: v3와 동일.

**v3 정정 — 필드명**: `판례정보일련번호` → **`판례일련번호`** (실제 API 응답)

**v3 보강 — 사건명 prefilter (양방향)** (`lib/lawApi.ts`):
```
DOMAIN_KEYWORDS (포함):
  아동학대·아동복지법·정서학대·신체학대·성적학대·유기·방임·체벌·교권

EXCLUDE_KEYWORDS (제외 — 명백히 도메인 밖):
  살해·치사·강제추행·유사강간·성착취·음행강요·그루밍·통신매체이용음란·촬영물·성폭력·성희롱
```

→ 4단계 확장 결과 중 *DOMAIN 포함 + EXCLUDE 미포함*인 사건만 통과. v3 알고리즘 단독으로는 형법 *교사범*(敎唆)·살해·성범죄 등 무관 판례 다량 섞임을 실증.

**4단계 확장**: v3 동일 (사건명 → 본문 search=2 → 2키워드 → 1키워드 안전망)

### 프롬프트 B-1 — 개별 판례 분석
- `analyzePrecedent()` (`lib/analyzePrecedent.ts`), Gemini 2.5 Flash
- 본문 추출: `extractRelevantBody()` — 판시사항 + 판결요지 + 주문만 (토큰 80-90% 절감, v3 그대로)
- **환각 방어 — 인용 검증**: `verifyQuotes()` — quote가 본문에 substring으로 있는지 normalize 후 매칭. 없으면 null. UI에 인용으로 노출 X
- **v3 보강 — 일반 법리·사용자 사안 구분**: 프롬프트에 "판례 일반 기준 단어를 사용자 사안에 단정 적용 금지" 명시
- 병렬 호출 (`Promise.all`, 최대 7건)

### 유사도 임계치 (v4 신규)
**`SIMILARITY_THRESHOLD = 30`%**: B-1 분석 후 유사도 30% 미만 판례는 결과에서 제외. 통과한 게 0건이면 노이즈라도 전체 표시 (빈 화면 회피).

### 프롬프트 B-2 — 종합 분석
- `summarizeReport()` (`lib/summarizeReport.ts`)
- **v3 보강**: 단정형 금지 + 판례 일반론과 사용자 사안 구분 + 좋은/나쁜 예시 명시

### Gemini 호출 재시도 (v4 신규)
**`withGeminiRetry()`** (`lib/gemini.ts`): `ECONNRESET`·`ETIMEDOUT`·`ECONNREFUSED` 등 일시 네트워크 오류 시 3회 재시도 (500ms·1s·1.5s 백오프). 모든 Gemini 호출 wrap.

### 법제처 fetch IPv4 강제 (v4 신규)
Windows + Node 24 환경에서 IPv6 fallback이 75초 hang을 유발. `dns.setDefaultResultOrder("ipv4first")` 모듈 진입 시 적용 (`lib/lawApi.ts`).

## 6. 환각 방어 — 다층 구조 (v4 강화)

1. **프롬프트 수준** — v3 동일 + B-2에 일반 법리/사용자 사안 구분 추가
2. **인용 검증** — `verifyQuotes()` 본문 substring 매칭 (공백·한국어 인용부호 normalize 후)
3. **결과 일관성** — 다층 LLM 호출 모순 패턴 학습 (riskFlag vs B-2 headline 모순) → riskFlag를 사용자 체크박스로 분리
4. **사용자에게 정직한 표시** — 모든 화면에 "법률 자문 아님" + AI 환각 차별점 명시
5. **사건명 prefilter** (v4) — 형법 교사범·성범죄·살해 등 차단
6. **유사도 임계치** (v4) — 30% 미만 노이즈 제거

### Gemini 환각 자체 검증 (2026-05-17)

`scripts/verify-hallucination.mjs`:
- 동일 Gemini Flash 2.5에 교사 사안 3개 직접 질문 → **인용 사건번호 8건 중 환각 6건 (75%)**
- 날짜·법원·사건명·본문 인용 모두 그럴듯하게 만들어져 사용자가 검증 못 함
- 든든샘은 법제처 실시간 호출이라 환각 0%
- → **결정적 차별점**. UI 곳곳에 명시 (랜딩 강조 박스 / 로딩 검증 카드 / 리포트 헤더 배지 / PrintHeader / footer)

## 8. 운영 모델 — 무료 + 책임 있는 한도 (Phase 1 쿠키 구현)

**원칙**: v3 동일. 모두에게 무료. 비용 폭주 방지 한도.

**구현** (`lib/rateLimit.ts`):
- **쿠키 기반** (`eduguard_usage`, httpOnly base64 JSON, 30일 슬라이딩 윈도우)
- 개인 일일 3회 / 월 10회 — v3 기본값 그대로
- 한도 초과 시 HTTP 429 + `LimitExceededView` (헌장 안내문 그대로 + `/support` 채널 유도)
- **한계**: 쿠키 지우면 reset. 의도적 우회는 Phase 2 (DB 도입) 시 차단
- 전체 일일 한도(`DAILY_GLOBAL_LIMIT`): 환경변수 정의만. 운영 단계 구현은 Phase 2

**진화의 문**: v3 동일.

## 10. 법적·윤리적 가드레일

v3 항목 동일 + 추가:
- **위험 신호 결정론적 판정** — 사용자 체크박스로만 판단. LLM이 *추측한 위험*은 사용자 신뢰 손상 위험이 커서 폐기 (다층 LLM 일관성 학습)
