# 교사 방패 프로젝트 — 한 페이지 요약 (v2)

> 작성일: 2026년 5월 17일 · 1차 개정
> 단계: 설계 완료, 개발 착수 직전
> 핵심 한 줄: **교사의 일상 언어로 아동학대 관련 판례를 찾아 일상 언어로 풀어주는, 모두에게 무료인 디지털 방패**

---

## 1. 왜 만드는가

학교 현장은 무분별한 고소·고발에 노출되어 있고, 교사에게는 *방패*가 없다. 변호사를 학교에 두는 것이 이상적이지만 현실적으로 어려우므로, 변호사의 핵심 역할 중 하나인 *"이 사안은 법적으로 어떤 의미인지 알려주는 것"* 을 AI 기반 도구로 대체한다.

핵심 사용자는 *아동학대 신고를 받았거나 받을 가능성에 직면한 교사*. 이들이 위기 상황에서 *3분 안에 자기 사안과 유사한 판례를 찾고, 법원의 판단 경향을 이해하고, 다음 행동을 결정할 수 있도록* 돕는다.

**개발 목적은 수익화가 아니라 교사들에게 실질적으로 도움이 되는 것이다.** 이 원칙이 모든 결정의 출발점이다.

---

## 2. 서비스 정체성

**우리는 법률 자문이 아니라 판례 탐색·정보 제공 서비스다.** 이 포지셔닝이 흔들리면 변호사법 109조 저촉 위험이 생긴다. 모든 화면·문구·면책 고지는 이 정체성을 일관되게 표현한다.

핵심 가치는 *"교사가 직접 찾기 어려운 판례를 쉽게 찾고 이해하도록 돕는 도구"*. 단정적 답을 주는 게 아니라 *법원이 어떻게 판단해왔는지의 경향을 보여주는 자료실*이다.

**무료라고 책임이 줄어들지 않는다.** 사용자가 비용을 지불하지 않더라도 정보의 정확성에 대한 기대는 동일하다. 환각 방어와 면책 고지는 *유료 서비스보다 더 철저하게* 적용된다.

---

## 3. 사용자 흐름

5개의 핵심 화면으로 구성된다.

1. **랜딩** — 신뢰 지표(누적 사용자·판례 수·익명성)와 진단 시작 CTA
2. **사안 입력** — 체크리스트 위주 + 자유 서술 보조. 3분 안에 입력 가능
3. **로딩** — 단계 표시(키워드 추출 → 검색 → 분석 → 정리)와 짧은 법률 상식
4. **분석 리포트** — AI가 정리한 판례 흐름 + 유사 판례 N건 + 대응 체크리스트
5. **판례 상세** — 개별 판례의 사실관계·판단 논리·결정문 핵심 문구

분기 화면이 두 개 더 있다.

- **위험 신호 화면** — 유죄 패턴 다수 또는 위험 키워드 감지 시. 단호하지만 따뜻한 톤으로 즉시 전문가 상담 권유
- **법률지원 채널 상세** — 각 기관의 운영시간·자격조건·연락처를 보여준 뒤 사용자 의사에 따라 `tel:` 링크로 발신

---

## 4. 데이터 구조

큐레이션을 도입하지 않기로 결정했으므로 데이터 구조는 크게 가벼워진다.

**유지되는 것**

- *태그 체계* — 7개 카테고리 약 85개 태그. 사용자 입력 분류와 LLM 키워드 변환에 활용
- *법률지원 채널 DB* — 전국 단위(1395, 교총, 전교조 본부) + 지역 단위(시도교육청, 16개 시도 교사노조, 전교조 지부). 모든 항목에 `last_verified_at` 필드로 검증일 관리
- *사용자 세션 로그* — 익명화된 통계만 저장. 자유 서술 본문은 분석 직후 즉시 폐기
- *사용량 한도 관리* — 일일·월간 분석 횟수 한도 체크용 카운터 (익명 식별자 기반)

**삭제된 것 (큐레이션 미도입 결정으로)**

- 판례 본체 저장 테이블
- 판례별 보호 논리 추출 테이블
- 판례 임베딩 검색 인덱스

판례 데이터는 *법제처 OPEN API에서 매번 실시간으로 가져오고*, 분석은 *LLM이 그 자리에서 수행*한다.

---

## 5. 핵심 기술 흐름

분석 1회는 다음 3단계로 구성된다.

**프롬프트 A — 키워드 변환**

사용자 입력(체크리스트 + 자유 서술)을 받아 3그룹의 검색 키워드(statute, action, context)와 위험 신호 여부, 1문장 요약을 JSON으로 반환한다. *법률 용어 매핑 표를 프롬프트에 명시*해 일관성 확보. Gemini 2.5 Flash로도 충분.

**법제처 API 호출 + 단계적 확장**

키워드를 AND로 조합해 호출하되, 결과량을 보고 *점진적으로 좁히거나 넓히는 4단계 전략*. 10건 이상 나오면 그 결과로 진행.

**프롬프트 B — 판례 본문 분석 (2단계)**

- *B-1*: 개별 판례를 한 건씩 LLM에 던져 유사도·결과·핵심 논리·사실관계·차이점·인사이트를 추출. 7건 병렬 호출. **모든 추출 정보에 본문 인용 필수**로 환각 방어.
- *B-2*: B-1 결과들을 모아 종합 분석. 헤드라인, 통계, 보호/위험 요인, 흐름 요약 본문 생성.

**모델 선택은 검증 후 결정**

기본은 Gemini 2.5 Flash로 운영. MVP 출시 전에 *같은 판례 10건을 Gemini와 Claude Sonnet에 똑같이 던져 결과 비교*. 정확도 차이가 크면 *B-1만 Claude로 가는 하이브리드* 도입 검토. 비용은 후술하는 한도 메커니즘으로 통제.

## 6. 환각 방어 — 다층 구조

법률 도메인에서 환각은 사용자에게 잘못된 안심을 주는 가장 위험한 오류다. *무료 서비스일수록 사용자가 더 무방비로 의존*하기 쉬워, 다층 방어를 더 철저히 한다.

1. **프롬프트 수준** — 본문 인용 필수, 확신 없으면 null, 본문에 없는 내용 금지
2. **인용 검증** — 서버에서 LLM 인용이 실제 본문에 있는지 문자열 매칭
3. **결과 일관성 체크** — B-1과 B-2 사이의 논리적 모순 자동 감지
4. **사용자에게 정직한 표시** — 모든 화면에 *"AI가 분석한 결과"* 라벨, *"법률 자문이 아님"* 면책
5. **운영 모니터링** — 사용자 신고 버튼으로 문제 케이스 수집해 프롬프트 개선

## 7. 비용 구조와 운영 한도

**분석 1회 예상 원가**

| 모델 구성 | 1회 원가 | 정확도 |
|---|---|---|
| 전부 Gemini 2.5 Flash | ₩200~400 | 검증 후 결정 |
| Flash + B-1만 Claude Sonnet (하이브리드) | ₩500~700 | 더 안전 |
| 사전 압축 + 캐싱 적용 시 | 추가 30~50% 절감 | 동일 |

**감당 가능한 운영 한도**

*"누군가는 비용을 부담해야 한다"* 는 사실을 직시한다. 본인이 부담 가능한 월 한도를 *미리 결정*하고, 그에 맞춰 사용량 시스템을 설계한다.

- *초기 부담 한도 예시*: 월 ₩100,000 (조정 가능)
- *이를 넘지 않기 위한 한도 구조*: 후술

---

## 8. 운영 모델 — 무료 서비스 + 책임 있는 한도

**원칙 — 모두에게 무료, 단 합리적 한도 내에서**

사용자는 비용을 지불하지 않는다. 다만 *비용 폭주를 방지*하고 *어뷰징을 차단*하기 위한 한도를 둔다. 한도에 도달한 사용자에게는 정중하게 안내한다.

**한도 설계 — 옵션 C 구현**

| 한도 종류 | 기본값 | 의미 |
|---|---|---|
| 개인 일일 한도 | 분석 3회 / 24시간 | 동일 익명 식별자 기준 |
| 개인 월간 한도 | 분석 10회 / 30일 | 동일 익명 식별자 기준 |
| 전체 일일 한도 | 분석 N회 / 24시간 | 전체 시스템 비용 통제 |

한도 도달 시 메시지: *"오늘의 무료 분석 한도가 소진되었습니다. 24시간 후 다시 이용하실 수 있습니다. 긴급한 사안이라면 아래 법률지원 채널을 바로 이용해주세요."* — 이렇게 안내하면서 *법률지원 채널 페이지로 자연스럽게 유도*. 한도가 차도 사용자가 *길을 잃지 않도록*.

익명 식별자는 브라우저 핑거프린트·세션 쿠키 조합으로 관리. *로그인 강제 없이* 한도 추적이 가능한 수준에서.

**진화의 문 — 옵션 D 구현**

처음의 무료 약속을 *깨는 것이 아니라 자연스럽게 진화*하기 위한 장치들.

1. **선택적 후원 버튼** — *"이 서비스가 도움이 되셨다면 후원해주세요"*. 카카오페이·토스 후원 링크. 강제하지 않고 가능성만 열어둠.
2. **사용량 폭주 시 안내** — 만약 운영 비용이 본인 한도를 초과하면, *"운영 지속을 위해 도움이 필요합니다"* 메시지로 후원 또는 자원 봉사 협력자 모집.
3. **학교·교원단체 협업 가능성** — 시범 학교나 교원단체에서 *"우리가 비용을 부담할 테니 우리 회원에게 한도 없이 제공해달라"* 같은 제안이 들어오면 검토. 단, 사용자에게는 *여전히 무료*.

## 9. 단계별 로드맵

**Phase 1 (0~3개월) — 작동하는 MVP**

- 큐레이션 없이 법제처 API + Gemini API 실시간 분석
- 화면 5+2개, 핵심 기능 완성
- *모델 정확도 비교 검증* — Gemini vs Claude 테스트
- 사용량 한도 시스템 구축
- 베타 사용자 100~300명 확보, 사용 패턴 데이터 수집

**Phase 2 (3~6개월) — 신뢰 구축**

- 사용자 피드백 반영한 프롬프트·검색 로직 튜닝
- 법률지원 채널 정보를 16개 시도 노조까지 보강
- *후원 채널 오픈* — 강제하지 않는 선택적 후원
- 월 활성 사용자 1,000명 돌파 목표

**Phase 3 (6~12개월) — 지속 가능성 확보**

- 시범 학교 1~2곳 무료 도입 사례 만들기
- 교원단체·시도교육청에 *"우리 회원/소속 교사에게 안정적 제공을 위한 후원"* 제안
- 운영 비용을 외부 자원으로 일부 충당하는 모델 시도
- 자연스럽게 교원단체 협업 제안이 들어오면 검토

**Phase 4 (1년 이후) — 큐레이션·파트너십 강화**

- 교원단체와 정식 데이터 파트너십
- 검증된 사례 100~200건 등록
- *"AI 분석 + 검증된 큐레이션 데이터"* 두 가치 결합
- 큐레이터·변호사 자문 비용을 교원단체·후원금으로 충당
- *사용자에게는 끝까지 무료* 원칙 유지

## 10. 법적·윤리적 가드레일

운영하면서 절대 놓치면 안 되는 원칙들.

- **변호사법 109조** — 우리는 정보 제공자, 법률 판단자 아님. 모든 화면·문구가 이 정체성 유지
- **개인정보보호** — 자유 서술 본문은 DB에 저장 안 함. 분석 직후 즉시 폐기. 익명 통계만 보관
- **사용량 한도 추적의 익명성** — 한도 관리용 식별자도 개인을 특정할 수 없게 설계
- **판례 저작권** — 판결문 원문은 공공 자산이라 활용 가능. 평석·해설은 인용 범위 내에서만
- **위험 신호 처리** — 신체학대 의심·반복성·부상 언급 등은 자동 분기. 즉시 전문가 상담 권유
- **정확성 책임** — 무료라도 책임이 줄지 않는다. 면책 고지는 작지 않게, 명확하게
- **무료의 정직함** — *"무료"* 라고 하면서 데이터 판매·광고 노출로 우회 수익화하지 않는다. 비용은 본인 부담·후원·기관 협업으로만 충당

**단기 (3~6주)**

- [ ] 5개 핵심 화면 구현

- [ ] 환각 방어 다층 구조 구현
- [ ] *사용량 한도 시스템 구축*

**중기 (6~10주)**

- [ ] 위험 신호 분기 화면 + 법률지원 채널 상세
- [ ] 익명 세션·로그 인프라
- [ ] *한도 도달 시 안내 흐름* — 법률지원 채널 자연스러운 연결
- [ ] 베타 출시 + 사용자 피드백 채널
- [ ] *작은 후원 버튼 도입* — 강제 없이

## 12. 한 줄로 다시

> *"위기 상황의 교사가 자기 사안과 비슷한 판례를 3분 안에 찾고 일상 언어로 이해할 수 있게 해주는, 모두에게 무료인 AI 기반 판례 탐색 도구. 합리적 사용 한도와 선택적 후원으로 지속 가능성을 확보한다."*

수익화는 목적이 아니다. 교사들에게 실질적 도움을 주는 것이 목적이다. 그러나 그 목적이 *지속될 수 있도록* 비용 통제 장치와 진화의 여지를 함께 둔다. 그것이 지금까지의 모든 설계가 가리키는 방향이다.
