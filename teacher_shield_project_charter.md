# 교사 방패 프로젝트 — 한 페이지 요약

> 작성일: 2026년 5월 17일
> 단계: 설계 완료, 개발 착수 직전
> 핵심 한 줄: **교사의 일상 언어로 아동학대 관련 판례를 찾아 일상 언어로 풀어주는, 학교가 사는 디지털 방패**

---

## 1. 왜 만드는가

학교 현장은 무분별한 고소·고발에 노출되어 있고, 교사에게는 *방패*가 없다. 변호사를 학교에 두는 것이 이상적이지만 현실적으로 어려우므로, 변호사의 핵심 역할 중 하나인 *"이 사안은 법적으로 어떤 의미인지 알려주는 것"* 을 AI 기반 도구로 대체한다.

핵심 사용자는 *아동학대 신고를 받았거나 받을 가능성에 직면한 교사*. 이들이 위기 상황에서 *3분 안에 자기 사안과 유사한 판례를 찾고, 법원의 판단 경향을 이해하고, 다음 행동을 결정할 수 있도록* 돕는다.

---

## 2. 서비스 정체성

**우리는 법률 자문이 아니라 판례 탐색·정보 제공 서비스다.** 이 포지셔닝이 흔들리면 변호사법 109조 저촉 위험이 생긴다. 모든 화면·문구·면책 고지는 이 정체성을 일관되게 표현한다.

핵심 가치는 *"교사가 직접 찾기 어려운 판례를 쉽게 찾고 이해하도록 돕는 도구"*. 단정적 답을 주는 게 아니라 *법원이 어떻게 판단해왔는지의 경향을 보여주는 자료실*이다.

---

## 3. 사용자 흐름

5개의 핵심 화면으로 구성된다.

1. **랜딩** — 신뢰 지표(누적 사용자·판례 수·익명성)와 진단 시작 CTA
2. **사안 입력** — 체크리스트 위주 + 자유 서술 보조. 3분 안에 입력 가능
3. **로딩** — 단계 표시(키워드 추출 → 검색 → 분석 → 정리)와 짧은 법률 상식
4. **분석 리포트** — AI가 정리한 판례 흐름 + 유사 판례 N건 + 대응 체크리스트
5. **판례 상세** — 개별 판례의 사실관계·판단 논리·결정문 핵심 문구

분기 화면이 두 개 더 있다.

- **위험 신호 화면** — 유죄 패턴 다수 또는 위험 키워드 감지 시. 단호하지만 따뜻한 톤으로 즉시 전문가 상담 권유
- **법률지원 채널 상세** — 각 기관의 운영시간·자격조건·연락처를 보여준 뒤 사용자 의사에 따라 `tel:` 링크로 발신

---

## 4. 데이터 구조

큐레이션을 도입하지 않기로 결정했으므로 데이터 구조는 크게 가벼워진다.

**유지되는 것**

- *태그 체계* — 7개 카테고리 약 85개 태그. 사용자 입력 분류와 LLM 키워드 변환에 활용
- *법률지원 채널 DB* — 전국 단위(1395, 교총, 전교조 본부) + 지역 단위(시도교육청, 16개 시도 교사노조, 전교조 지부). 모든 항목에 `last_verified_at` 필드로 검증일 관리
- *사용자 세션 로그* — 익명화된 통계만 저장. 자유 서술 본문은 분석 직후 즉시 폐기

**삭제된 것 (큐레이션 미도입 결정으로)**

- 판례 본체 저장 테이블
- 판례별 보호 논리 추출 테이블
- 판례 임베딩 검색 인덱스

판례 데이터는 *법제처 OPEN API에서 매번 실시간으로 가져오고*, 분석은 *LLM이 그 자리에서 수행*한다.

---

## 5. 핵심 기술 흐름

분석 1회는 다음 3단계로 구성된다.

**프롬프트 A — 키워드 변환**

사용자 입력(체크리스트 + 자유 서술)을 받아 3그룹의 검색 키워드(statute, action, context)와 위험 신호 여부, 1문장 요약을 JSON으로 반환한다. *법률 용어 매핑 표를 프롬프트에 명시*해 일관성 확보. 비싼 모델까지 갈 필요 없이 Haiku급으로도 충분.

**법제처 API 호출 + 단계적 확장**

키워드를 AND로 조합해 호출하되, 결과량을 보고 *점진적으로 좁히거나 넓히는 4단계 전략*. 10건 이상 나오면 그 결과로 진행.

**프롬프트 B — 판례 본문 분석 (2단계)**

- *B-1*: 개별 판례를 한 건씩 LLM에 던져 유사도·결과·핵심 논리·사실관계·차이점·인사이트를 추출. 7건 병렬 호출. **모든 추출 정보에 본문 인용 필수**로 환각 방어. Sonnet급 모델 권장.
- *B-2*: B-1 결과들을 모아 종합 분석. 헤드라인, 통계, 보호/위험 요인, 흐름 요약 본문 생성. Haiku급 가능.

이 모든 출력이 리포트 화면의 데이터가 된다.

## 6. 환각 방어 — 다층 구조

법률 도메인에서 환각은 사용자에게 잘못된 안심을 주는 가장 위험한 오류다. 5층 방어로 막는다.

1. **프롬프트 수준** — 본문 인용 필수, 확신 없으면 null, 본문에 없는 내용 금지
2. **인용 검증** — 서버에서 LLM 인용이 실제 본문에 있는지 문자열 매칭
3. **결과 일관성 체크** — B-1과 B-2 사이의 논리적 모순 자동 감지
4. **사용자에게 정직한 표시** — 모든 화면에 *"AI가 분석한 결과"* 라벨, *"법률 자문이 아님"* 면책
5. **운영 모니터링** — 사용자 신고 버튼으로 문제 케이스 수집해 프롬프트 개선

---

## 7. 비용 구조

분석 1회 원가는 모델 선택에 따라 다르다.

- *전부 Haiku*: 약 ₩80~120 (환각 위험 큼)
- *전부 Sonnet*: 약 ₩800~1,200 (정확도 가장 안전)
- *하이브리드 (A·B-2 = Haiku, B-1 = Sonnet)*: 약 ₩700~900
- *판례 본문 사전 압축 + 하이브리드*: 약 ₩300~500
- *부분 캐싱 적용 시*: 추가 30~50% 절감

법률 도메인의 정확도 요구를 고려해 **B-1은 Sonnet 권장**. 비용 최적화는 압축·캐싱·하이브리드로 푼다.

---

## 8. 비즈니스 모델 — 학교 단위 연간 구독

이 프로젝트의 가장 본질적인 결정. 개인 결제가 아니라 *학교가 사는 디지털 방패*로 포지셔닝한다.

**왜 학교 구독인가**

- 위기 상황 교사에게 결제 마찰 0
- 학교 행정의 자연스러운 구매 동기 (교사 보호 책임 이행 증명)
- 평균 사용량 분산으로 비싼 모델 사용해도 단위 경제 건전
- 학교 1곳 도입 = 교사 수십~수백 명 노출

**가격 구조 (출발점, 검증 후 조정)**

| 티어 | 학교 규모 | 연 구독료 | 교사 1인당 |
|---|---|---|---|
| 1 | 30명 이하 | ₩300,000 | 약 ₩10,000 |
| 2 | 30~70명 | ₩600,000 | 약 ₩9,000 |
| 3 | 70~150명 | ₩1,000,000 | 약 ₩7,000 |
| 4 | 150명+ | ₩1,500,000+ | 협의 |

**병행 모델**

학교 영업 사이클이 길기 때문에 *개인 결제 모델을 병행*해 현금 흐름을 만들고, *개인 사용자가 자기 학교에 추천하는 흐름*을 영업 입구로 활용한다.

**접근성 안전망**

- 교원단체 회원에게 무료 코드 발급
- 경제적 어려움 호소 시 무료 제공
- 시범 학교에는 3개월 무료 운영

## 9. 단계별 로드맵

**Phase 1 (0~3개월) — 작동하는 MVP**

- 큐레이션 없이 법제처 API + LLM 실시간 분석
- 화면 5+2개, 핵심 기능 완성
- 베타 사용자 100~300명 확보, 사용 패턴 데이터 수집

**Phase 2 (3~6개월) — 신뢰 구축**

- 사용자 피드백 반영한 프롬프트·검색 로직 튜닝
- 법률지원 채널 정보를 16개 시도 노조까지 보강
- 시범 학교 1~2곳 무료 운영
- 월 활성 사용자 1,000명 돌파 목표

**Phase 3 (6~12개월) — 학교 영업 본격화**

- 시범 학교 사례를 기반으로 다른 학교 영업
- 가격·티어·계약서·영업 자료 정식 셋업
- 시도교육청 단위 협약 시도
- 자연스럽게 교원단체 협업 제안이 들어오면 검토

**Phase 4 (1년 이후) — 큐레이션·파트너십 강화**

- 교원단체와 정식 데이터 파트너십
- 검증된 사례 100~200건 등록
- *"AI 분석 + 검증된 큐레이션 데이터"* 두 가치 결합
- 큐레이터·변호사 자문 비용을 교원단체·후원금으로 충당

## 10. 법적·윤리적 가드레일

운영하면서 절대 놓치면 안 되는 원칙들.

- **변호사법 109조** — 우리는 정보 제공자, 법률 판단자 아님. 모든 화면·문구가 이 정체성 유지
- **개인정보보호** — 자유 서술 본문은 DB에 저장 안 함. 분석 직후 즉시 폐기. 익명 통계만 보관
- **판례 저작권** — 판결문 원문은 공공 자산이라 활용 가능. 평석·해설은 인용 범위 내에서만
- **위험 신호 처리** — 신체학대 의심·반복성·부상 언급 등은 자동 분기. 즉시 전문가 상담 권유
- **정확성 책임** — 유료 서비스는 책임이 더 크다. 면책 고지는 작지 않게, 명확하게

## 11. 다음 작업 우선순위

이 한 페이지가 모든 결정의 기준이다. 여기서 출발해 다음 작업으로 들어간다.

**즉시 (1~2주)**

- [ ] 법제처 OPEN API 신청·인증 받기
- [ ] 기술 스택 결정 (Next.js + Vercel + Supabase 등 후보)
- [ ] 도메인·기본 인프라 셋업
- [ ] 프롬프트 A·B 초안 작성 및 테스트

**단기 (3~6주)**

- [ ] 5개 핵심 화면 구현
- [ ] 법제처 API 연동 + 단계적 확장 로직
- [ ] LLM 호출 파이프라인 (병렬 처리, 캐싱)
- [ ] 환각 방어 다층 구조 구현

**중기 (6~10주)**

- [ ] 위험 신호 분기 화면 + 법률지원 채널 상세
- [ ] 익명 세션·로그 인프라
- [ ] 베타 출시 + 사용자 피드백 채널
- [ ] 첫 시범 학교 컨택

## 12. 한 줄로 다시

> *"위기 상황의 교사가 자기 사안과 비슷한 판례를 3분 안에 찾고 일상 언어로 이해할 수 있게 해주는, 학교가 사는 AI 기반 판례 탐색 도구."*

큐레이션은 미루고, 가장 작은 형태로 빠르게 만들어, 사용자에게서 가치를 증명받은 뒤, 학교와 교원단체로 확장한다. 그것이 지금까지의 모든 설계가 가리키는 방향이다.
