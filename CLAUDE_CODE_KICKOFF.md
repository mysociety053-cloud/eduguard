## 프로젝트 개요

**프로젝트명**: EduGuard (교사방패)  
**목적**: 아동학대 신고에 직면한 교사가 자기 사안과 유사한 판례를 쉽게 찾고 일상 언어로 이해할 수 있게 돕는, 모두에게 무료인 AI 기반 판례 탐색 도구  
**기술 스택**: Next.js 15+ (App Router, TypeScript) + Tailwind CSS + Gemini 2.5 Flash API + 법제처 OPEN API  
**배포 목표**: Vercel  

## 내가 이미 가지고 있는 것

1. **법제처 OPEN API 인증값(OC)**: `eduguard` — 승인 완료, 판례·법령·해석례 등 모두 활용 가능
2. **Gemini API 키**: 발급 완료 (`.env.local`에 저장 예정)
3. **검색 전략 검증 완료**: 본문 검색(`search=2`) + 3 키워드 AND 조합이 가장 정밀한 매칭을 줌
4. **LLM 프롬프트 검증 완료**: Gemini 2.5 Flash로 무죄·유죄 판례 모두 정확히 분석 가능, 환각 0건
5. **헌장에 정리된 화면 설계 5+2개**: 랜딩, 사안 입력, 로딩, 분석 리포트, 판례 상세 + 위험 신호 화면, 법률지원 채널 상세

## 첫 번째로 해줬으면 하는 일

다음 순서로 진행해줘.

### 1단계 — 헌장 정독 후 확인 질문

`PROJECT_CHARTER.md`를 읽고, 시작하기 전에 명확히 해야 할 점이 있다면 나에게 1~3가지 질문으로 정리해서 물어봐. 예를 들어:

- 화면 디자인의 톤 (차분한 vs 친근한 vs 전문적)
- 첫 MVP에 포함할 화면 우선순위 (5+2 전부 vs 핵심 3개부터)
- 데이터베이스 사용 여부 (MVP 초기에는 무필요로 설계됐지만 사용량 한도 추적이 필요)

### 2단계 — 환경 점검

내 컴퓨터에 다음이 설치되어 있는지 확인하는 명령을 알려줘:
- Node.js (버전 20 이상 권장)
- npm 또는 pnpm
- Git

각각의 설치 확인 명령과, 만약 없다면 어떻게 설치해야 하는지 안내해줘. 내가 결과를 알려주면 다음 단계로 가자.

### 3단계 — 프로젝트 초기 구조 생성

환경 점검이 끝나면:
- Next.js 15+ (App Router, TypeScript, Tailwind CSS) 프로젝트 생성
- 기본 폴더 구조 잡기 (`/app`, `/components`, `/lib`, `/types` 등 — 헌장에 맞게)
- `.env.local` 템플릿 (값은 비워두고 키만)
- `.gitignore` 설정 (특히 `.env.local`, `node_modules` 포함)
- `package.json`에 기본 스크립트 정리

### 4단계 — 가장 작은 작동하는 첫 화면

여러 기능을 한 번에 만들지 말고, **가장 작은 작동하는 첫 화면**부터.

추천: **사안 입력 화면 → 로딩 → 분석 리포트** 까지 한 줄로 동작하는 최소 흐름. 단, 처음엔 *법제처 API와 Gemini를 실제 호출*하지 말고, *목업 데이터로 흐름이 작동*하는지부터. 그게 끝나면 실제 API 연동.

## 작업 원칙

1. **헌장을 기준으로** — 모든 결정은 `PROJECT_CHARTER.md`와 일치해야 함. 헌장과 다른 결정이 필요하면 *나에게 먼저 확인*해줘.
2. **작게 자주** — 한 번에 큰 변경 X. 작은 단위로 만들고 *나에게 확인 후 다음*.
3. **이유를 설명** — 새 라이브러리 추가, 파일 구조 변경 등은 *왜 그렇게 하는지* 짧게 설명해줘.
4. **변호사법 109조 의식** — 우리는 *법률 자문이 아니라 판례 정보 제공*. 모든 화면 텍스트에 이 정체성 반영.
5. **무료 운영의 정직함** — 광고 코드, 트래킹 픽셀, 데이터 수집 같은 건 *절대* 추가하지 않음.
